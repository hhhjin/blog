컴포넌트 UI를 보여주는 방법은 유저가 작업하고 있는 개발 디렉토리를 앱 데이터 폴더에 복사하고 dev 서버를 실행한 후 iframe으로 보여주는 방법을 택했었다.

그런데 초기에 이 방법을 간단하게 구현해보면서 테스트를 해봤을 때 데스크탑 앱을 종료했음에도 불구하고 개발 서버가 죽지 않는 상황이 발생했었다.

```typescript
import { Command } from "@tauri-apps/api/shell";

const child = new Command("pnpm", ["--dir", appDataDirPath, "dev"]).spawn();
```

이런 방식으로 프로세스를 만들면 `pnpm -> next dev -> next-router-어쩌구` 이런 식으로 하위 프로세스여러 개 생겨나는데 `child.kill()`을 하면 처음에 생성한 `pnpm` 프로세스만 죽고 그 자식들은 살아있다. 이걸 해결하는 작업을 했다.

일단 이 원인은 생성한 프로세스가 죽어도 그 자식 프로세스는 동작하게 하는 사용자의 의도가 있을 수 있어서 운영체제나 Tauri에서는 그 책임을 생성한 프로세스에게 떠넘기고 생성한 프로세스만 종료하게 되는 거라고 한다. 그래서 다른 방법을 생각하기로 했다.

다른 방법으로는 개발 서버가 사용하고 있는 포트 번호를 알아내서 프로세스를 종료시키는 방법이 있다. 어차피 iframe 방법을 사용하려면 포트 번호를 알아야 하기 때문에 이 방법은 필요했다.

내가 쓰고 있는 유닉스 환경에서는 커맨드 창에서 `lsof -i :3000`으로 프로세스 ID를 얻고 `kill PID` 같이 수동으로 종료를 시킬 수 있다. 이 방법을 rust로는 https://github.com/ZingerLittleBee/port-killer-rs 을 사용해서 처리했다.

이제 문제는 포트 번호를 알아내는 방법이다. 만약 유저가 Next.js를 쓰고 있다면 기본 포트 번호는 3000이다. 그럼 앱을 종료할 때 Next.js를 사용하고 있으니까 3000번 포트를 사용하고 있는 프로세스를 죽이면 될까? 그렇지만 이미 3000을 사용하고 있는 상태에서 next dev를 실행했다면 3001번이 사용되기 때문에 이 사실을 모른 채로 멀쩡한 다른 프로세스를 죽일 수도 있게 된다. 또는 사용자가 임의로 `"dev": "next dev -p 3014"`같은 설정을 `package.json`에 해두었다면 또 문제가 발생한다. 물론 `package.json`을 파싱해서 포트번호를 알아낼 수는 있겠지만 이것도 좋은 방법은 아닌 것 같다.

그래서 그냥 앱에서 직접 포트 번호를 설정해서 개발 서버를 실행하는 방식을 생각했다. 예를 들어 위에서 작성했던 `Command`를 사용한 코드에서 `pnpm dev -p PORTNUMBER`를 실행하게 하면 next.js의 경우 내가 설정한 포트 넘버로 서버를 실행할 것이다. 예외로 이미 `package.json`에 포트를 설정해놨대도 이 방식을 사용하면 Next.js의 경우 값을 덮어씌워서 잘 동작하는 것까지는 확인했다.

그럼 포트 넘버를 어떻게 정할까? 만약 포트 넘버를 정하고 개발 서버를 실행했는데 이전과 같은 경우로 이미 사용 중이면 포트가 바뀐 상태로 실행된다. 방법은 개발 서버를 실행하기 전에 사용할 포트를 누군가가 점유하고 있는지 확인하면 된다.

```rust
fn is_available_port(port: u16) -> bool {
    // bind로 생성되는 값을 사용하지 않으면 자동으로 socket close
    match TcpListener::bind(("127.0.0.1", port)) {
        Ok(_) => true,
        Err(_) => false,
    }
}
```

이런 함수를 이용하면 간단하게 해결할 수 있다.

### 정리

1. 앱을 종료해도 좀비 프로세스(개발 서버)가 남아있다.
2. 포트 번호를 이용해서 종료하는 방법을 고안함.
3. 그러려면 포트 번호를 알아야 함.
4. 포트 번호를 알아내기보다는 포트를 직접 설정하는 방법을 고안함.
5. 사용하고 있지 않은 포트를 알아내서 개발 서버를 실행함.
6. 사용된 포트 번호를 `<iframe src="http://localhost:port" />`에 사용해서 컴포넌트 UI를 보여준다.
