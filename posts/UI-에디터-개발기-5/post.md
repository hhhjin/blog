4편에서 계획했던 건 하다가 중간에 취소했다. 우선 데이터를 어떻게 다룰지 다시 생각해보는 게 좋을 것 같았다.

처음 계획했던 건 우선 로컬 sqlite같은 저장소에 데이터를 저장하는 방식을 택한 후 개발을 진행하고 나중에 서버에 동기화하는 작업을 추가하려고 했는데 이 방법은 너무 복잡할 것 같아서 그냥 서버에 데이터를 저장하는 방식으로 전환했다. 그 대신 offline에도 동작할 수 있게 하려고 한다.

일단 서버 데이터 관리는 zustand 대신 Tanstack query를 사용하기로 마음먹었고 그 후 서버 코드를 작성하기 시작했다. 오늘은 인증 부분을 구현했고 간략하게 어떤 걸 했는지 적어보겠다.

- Cloudflare Workers
- D1, Drizzle ORM
- Lucia
- Testing

### Cloudflare Workers

서버리스 플랫폼이다. 원래는 프록시용으로 많이 쓰던 건데 이제는 기능이 많이 추가돼서 범용적으로 사용할 수 있다. 무료 플랜이 괜찮고 인프라 관리도 편해서 선택했다. 참고로 글로벌 엣지 서버에서 동작한다.

### D1, Drizzle ORM

D1은 Cloudflare Workers 내부에 껴있는 서버리스 SQL 데이터베이스다. 이것도 Workers와 마찬가지로 가격이 합리적이고 내년 초에 Beta 딱지를 뗄 것 같다. 나중에 글로벌 복제 기능이 추가될 텐데 이게 무료라고 한다.

Drizzle ORM은 SQL 쿼리를 생성해주는 라이브러리이다. ORM이지만 최대한 SQL문과 비슷하게 코드를 작성할 수 있게 한다. 같이 딸린 Drizzle kit이라는 애는 마이그레이션 파일을 생성해준다.

### Lucia

Next.js 대체제 라이브러리라고 예전에 어디선가 봐서 기억해뒀다가 이제서야 사용해봤다. Next.js를 쓰면 편하지만 Vercel 서버리스에서 D1을 사용하려면 번거로워서 선택했다. 지금은 굳이 D1을 쓰고 싶다.

그래도 쓸만한 것 같다. 먼가 나중에 유지보수가 안 될것 같은 싸한 느낌이 들지만 내부 동작도 간단해서 포크하거나 단순히 OAuth용으로도 써도 괜찮은 것 같다.

### Testing

Wokers의 개발 환경은 아직 안정기가 아니라서 Deprecated된 기능이 많고 문서가 부실하다. Workers 환경에서 테스팅 방법에 대해 좀 더 연구해보고 남들에게 도움이 될 만한 글을 조만간 써봐야겠다.

### 그 외

요즘 너무 빠르게 개발하는 거에 초점을 맞추다 보니 무언가 놓치고 있다는 느낌이 든다. 아직은 깊이보다는 좌우로 넓게 펼치는 중이라 그런지 마음이 급하다.
