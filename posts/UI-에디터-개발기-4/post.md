### 1. 작업 폴더 복사

사용자 작업 환경과 동일하게 하기 위해서 작업 폴더 전체를 앱 데이터 폴더로 복사하는 아이디어를 냈었다. Tauri의 자바스크립트 api에는 파일 복사 함수만 제공하기 때문에 Rust 코드로 구현해야 한다. 일단은 [fs_extra](https://docs.rs/fs_extra/latest/fs_extra/index.html)를 사용해서 구현할 예정이다.

근데 복사하는 방법을 이용하면 에디터의 작업 환경이 유저 개발 환경과 동일하다는 조건이 깨질 수가 있다. 예를 들어 원본 `package.json`내용이 달라지는 경우이다. 그래서 이 점을 보완해야 하고 .git을 추적하는 아이디어가 있다.

### 2. 캔버스 컴포넌트 복제 환경에 삽입

캔버스 컴포넌트(`<Canvas />`)는 그냥 사용자 컴포넌트를 렌더링해서 보여주는 역할이다. 이 컴포넌트를 복제된 환경의 엔트리 파일에 삽입해야 하는데 Next.js의 경우 `app/page.tsx` 또는 `pages/index.tsx`가 `<Canvas />`를 반환하게 바꿔줘야 한다.

함수 인터페이스: insertCanvasComponent(jsx: string)

사용자에게 프로젝트를 생성하기 전 엔트리 파일 경로를 입력받는 게 좋겠다. 나중에는 프레임워크 별 엔트리 포인트를 미리 저장해놓거나 package.json의 main값을 보고 자동으로 알아낼 수도 있을 듯.

참고로 위 두 작업은 제일 처음 프로젝트를 생성할 때 실행 해야 한다.

### 3. 복제 환경 개발 서버 실행

복사한 환경에서 개발 서버를 실행해서 메인 화면의 iframe에 띄어야 한다. 개발을 시작하기 전에 했던 검증 방법을 이용해서 프로세스를 실행하면 될 것 같다.

### 4. 컴포넌트 렌더링

이전에 만들었던 컴포넌트 리스트에서 컴포넌트 항목을 클릭하면 프레임 생성 모달을 띄운다. 프레임은 `width`와 `height`값과 컴포넌트의 `props`를 입력받는다. 그럼 복제된 폴더의 `<Canvas />`컴포넌트 내부에 `<Frame width={200} height={200}><Button>나는 버튼입니다</Button></Frame>`이런 코드를 `import`문과 함께 삽입한다. 그러려면 `Button`의 상대경로를 알아야 할 테다.

이 작업이 아마 제일 복잡한 과정 중에 하나다. 사실상 여기까지만 하면 초기에 컴포넌트 보여주는 것까지 계획했던 게 마무리될 것 같다.

### 5. 데이터 저장

지금껏 해왔던 작업 중에 데이터베이스를 사용해서 저장하거나 불러온 건 없었다. 실제로 사용할 수 있는 제품이 되기 위해서는 사용자가 작업한 내용이 계속 어딘가에 저장되어야 한다. 애초에 생각했던 건 로그인 없이 사용 가능한 로컬에서 동작하는 데스크탑앱이었는데 데이터베이스 서버를 두어야 할지 고민이다. 이 고민은 나중에 하기로.
