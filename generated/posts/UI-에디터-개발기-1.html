일단 지금 원하는 앱 기능은 왼쪽에 파일 목록이 있고 파일을 클릭하면 화면 중앙에 컴포넌트 렌더링 된 거 보여주고 오른쪽에 코드 보여주는 정도이다. 여기까지 최우선 목표로 하고 마우스로 디자인하는 건 다음에 생각해 봐도 될 것 같다.

그럼, 관심사를 나눠보자. 크게 3가지로 나눠볼 수 있을 것 같은데 위에서 정의한 기능으로 나누면 될 것 같다.

1. 파일 트리? 뷰어?

뭐라고 불러야 할지 애매한데 그냥 트리로 부르겠다. 요놈은 파일 목록을 보여주고 fs 작업을 한다. 제일 처음 사용자가 파일을 클릭하면 관련 데이터를 오른쪽으로 보내주는 게 가장 큰 역할이다.

2. 프레임

사용자가 파일 트리의 특정 파일을 클릭하면 그 파일 속 컴포넌트를 렌더링하는 놈이다. 근데 생각해 보니 스토리북같은 경우에는 파일을 작성해서 기본 props를 넣어줬던 것 같은데 얘는 지정한 props가 없는데 어떻게 해야 할까? 앱 내에서 props 설정하고 컴포넌트를 추가하는 작업이 필요하다.

3. 위에서 추가된 컴포넌트 생성 작업

다시 1번으로 돌아가서 파일을 클릭하면 바로 오른쪽에 패널이 나오면서 프레임 내부에 렌더링할 수 있는 컴포넌트 목록을 보여준다. 그리고 프레임으로 드래그 앤 드랍을 하면 프레임에 데이터 보내는 걸로.

4. 다시 프레임

컴포넌트 데이터를 왼쪽에서 받으면 props 입력받고 렌더링까지 하는 역할

4. 제일 오른쪽 패널

이건 아직 컴포넌트 수정 기능이 없어서 쓸모는 없지만 일단은 파일 코드를 보여주는 정도까지만 만들어놔도 될 것 같다.
